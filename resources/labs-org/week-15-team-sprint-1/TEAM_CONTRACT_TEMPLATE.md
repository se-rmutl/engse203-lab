## กติกาการทำงานร่วมกัน

- ใช้ issue เพื่อแจกแจงงาน
- พัฒนา feature ใน branch ของตนเอง
- ใช้ pull request ก่อน merge เข้า `main`
- ทุกคนต้องมี commit และหลักฐานการมีส่วนร่วมที่ตรวจสอบได้
- สื่อสารปัญหาและความเสี่ยงภายในช่องทางทีมที่ตกลงกัน

## Definition of Done

- [ ] Feature ทำงานตาม acceptance criteria
- [ ] มีการทดสอบ/ทดสอบด้วยตนเองตาม checklist
- [ ] README หรือ API contract ได้รับการปรับปรุง
- [ ] ไม่มี secret หรือไฟล์ไม่จำเป็น
- [ ] Pull request ได้รับการ review ตามข้อตกลงทีม
