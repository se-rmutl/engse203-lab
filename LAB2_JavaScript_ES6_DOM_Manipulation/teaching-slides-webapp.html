HTML5 Semantic Elements

ความหมายของ Semantic HTML

Semantic HTML คือการใช้ HTML elements ที่มีความหมายชัดเจนเพื่ออธิบายเนื้อหา

❌ Non-semantic Elements
<div class="header"> <div class="navigation">...</div> </div> <div class="main-content"> <div class="article">...</div> </div>
✅ Semantic Elements
<header> <nav>...</nav> </header> <main> <article>...</article> </main>
  • ช่วยให้ Search Engines เข้าใจเนื้อหาดีขึ้น
  • เพิ่มการเข้าถึงสำหรับผู้พิการ
  • โค้ดอ่านง่าย และ maintain ง่าย
  • มาตรฐาน HTML5

HTML5 Semantic Elements หลัก

<header>

ส่วนหัวของเว็บไซต์หรือ section

<nav>

เมนูนำทางหลัก

<main>

เนื้อหาหลักของหน้า

<section>

ส่วนของเอกสารที่มีหัวข้อ

<article>

เนื้อหาที่สมบูรณ์อิสระ

<aside>

เนื้อหาด้านข้าง sidebar

<footer>

ส่วนท้ายของเว็บไซต์

<address>

ข้อมูลติดต่อ

<time>

วันที่และเวลา

💡 เคล็ดลับ

ใช้ semantic elements ตามความหมายจริง ไม่ใช่ตามรูปลักษณ์

โครงสร้าง HTML5 ที่ถูกต้อง

HTML5 Document Structure
<!DOCTYPE html> <html lang="th"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>ชื่อหน้าเว็บ</title> </head> <body> <header> <nav>เมนูนำทาง</nav> </header> <main> <section> <h1>หัวข้อหลัก</h1> <article>เนื้อหาบทความ</article> </section> <aside>เนื้อหาด้านข้าง</aside> </main> <footer>ส่วนท้าย</footer> </body> </html>

กฎสำคัญ:

  • ใช้ <h1> เพียงอันเดียวในหน้า
  • Heading hierarchy: h1 → h2 → h3 → h4 → h5 → h6
  • <main> ใช้เพียงอันเดียวในหน้า
  • เขียน alt text ที่มีความหมายสำหรับรูปภาพ

Web Accessibility

ทำไมต้องสนใจ Accessibility?

  • ช่วยให้คนพิการเข้าถึงเว็บไซต์ได้
  • ปรับปรุง SEO และ search rankings
  • เพิ่มผู้ใช้และลูกค้า
  • กฎหมายในหลายประเทศกำหนดให้ปฏิบัติ
  • เป็นการออกแบบที่ดีสำหรับทุกคน

หลักการพื้นฐาน

Accessibility Best Practices
<!-- ✅ Alt text ที่มีความหมาย --> <img src="chart.png" alt="กราฟแสดงยอดขายเพิ่มขึ้น 25% ในปี 2025"> <!-- ✅ Form labels --> <label for="email">อีเมล</label> <input type="email" id="email" name="email" required> <!-- ✅ ARIA attributes --> <button aria-label="ปิดหน้าต่าง">×</button>

Flexbox Basic Setup
.container { display: flex; justify-content: space-between; /* จัดแนวแกนหลัก */ align-items: center; /* จัดแนวแกนตั้งฉาก */ gap: 1rem; /* ระยะห่างระหว่าง items */ } .item { flex: 1; /* ขยายเต็มพื้นที่ */ flex-shrink: 0; /* ไม่หดตัว */ }
  • Main Axis: แกนหลัก (horizontal หรือ vertical)
  • Cross Axis: แกนตั้งฉาก
  • Flex Container: parent element
  • Flex Items: child elements

Feature Flexbox Grid

CSS Animations

Transitions vs Animations

CSS Transitions
.button { background: blue; transition: all 0.3s ease; } .button:hover { background: red; transform: scale(1.1); }
CSS Keyframes
@keyframes slideIn { 0% { transform: translateX(-100%); opacity: 0; } 100% { transform: translateX(0); opacity: 1; } } .element { animation: slideIn 0.5s ease-out; }

Animation Demo (hover เพื่อหยุด)

  • Transitions: สำหรับการเปลี่ยนแปลง 2 สถานะ
  • Animations: สำหรับการเคลื่อนไหวที่ซับซ้อน
  • Performance: ใช้ transform และ opacity
  • Timing: ease, linear, cubic-bezier

Mobile-First Media Queries
/* Mobile First (320px+) */ .container { width: 100%; padding: 1rem; } /* Tablet (768px+) */ @media (min-width: 768px) { .container { max-width: 750px; margin: 0 auto; } } /* Desktop (1024px+) */ @media (min-width: 1024px) { .container { max-width: 1200px; padding: 2rem; } }

เทคนิคสำคัญ

  • Viewport Meta Tag: <meta name="viewport" content="width=device-width">
  • Flexible Images: max-width: 100%
  • Fluid Grids: ใช้ % แทน px
  • Breakpoints: 576px, 768px, 992px, 1200px

CSS Preprocessors (Sass/SCSS)

ทำไมต้องใช้ Preprocessors?

  • Variables สำหรับค่าที่ใช้ซ้ำ
  • Nesting เพื่อโครงสร้างที่ชัดเจน
  • Mixins สำหรับโค้ดที่ใช้ซ้ำ
  • Functions และ Operations
  • Modular CSS ด้วย Partials
SCSS Variables & Nesting
$primary-color: #3498db; $font-size-base: 1rem; .navbar { background: $primary-color; padding: 1rem; ul { list-style: none; li { display: inline-block; a { color: white; text-decoration: none; &:hover { color: lighten($primary-color, 20%); } } } } }
SCSS Mixins
@mixin button($bg-color, $text-color: white) { background: $bg-color; color: $text-color; border: none; padding: 0.5rem 1rem; border-radius: 4px; cursor: pointer; &:hover { background: darken($bg-color, 10%); } } .btn-primary { @include button($primary-color); } .btn-secondary { @include button(#6c757d); }

Best Practices

HTML Best Practices

  • ใช้ semantic elements ตามความหมาย
  • เขียน alt text ที่อธิบายเนื้อหา
  • รักษา heading hierarchy
  • ใส่ form labels ทุกตัว
  • Validate HTML markup

CSS Best Practices

  • Mobile-first responsive design
  • ใช้ Flexbox สำหรับ 1D layouts
  • ใช้ Grid สำหรับ 2D layouts
  • Optimize animations (transform, opacity)
  • ใช้ CSS variables สำหรับ theming

Performance Tips

  • Minimize CSS file size
  • ใช้ lazy loading สำหรับรูปภาพ
  • Critical CSS inlining
  • Avoid layout-triggering properties
  • Use hardware acceleration
